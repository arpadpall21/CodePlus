Description and Demonstration :

- OAuth 2.0 is an Authorization protocol meaning we can do certain stuff in the behalf of a user but we don't know anyting about the user's credentials

- OAuth 2.0 terminology:

- Resource Onwer: User who owns the resource

- Authorization Server: Server that deals with all authorizations

- Resource Server: Server holding the resource owner's resources

- Authorization Grant: Authorization code that exchangable for Access & Refresh tokens (Represents the resource owner's authorization)

- Redirect URI: URL where we want to recevies the Authorization Grant, Access and Refresh Tokens (called by the auth server on successfull resource owner access)

- Authorization Scope: Limited authorization categories (scopes) defined by the 3rd party app

- Response Type: the OAuth type we whant to receive (code = Authorization Code / token = Access Token)

Open Id Connect (OIDC)

- Open Id Connect is an Authentication protocol meaning we ask a trusted authentication provider to authenticate a user for us

- when sending the redirect URI request we specify openid as scope (is that simple)

- when exchanging the Authorization code for an Access token we also get an Id token (JWT token containing the user's id)
