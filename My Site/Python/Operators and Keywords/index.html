Arithmetic Operators
? nrType + nrType - addition
? nrType - nrType - subtraction
? nrType * nrType - multiplication
? nrType / nrType - division
? nrType ** nrType - exponentiation
? nrType % nrType - modulo division (divides integers and returns the reminder)
? nrType // nrType - floor division the division quotient is floored to the closest integer
Unary Operators
? +var - unary plus, (var must contain a number type) returns the value held by the variable (does nothing)
? -var - unary minus, (var must contain a number type) flips the sign of the number type and returns the value held by the variable
- for complex types both real and imaginary signs are flipped
Assignement Operators
? var = val:any - assigns (binds) val to the variable
? var += nrType - addition assignement, adds var to nrType then reassigns the result to var (same as var = var + nrType)
? var -= nrType - subtraction assignement, subtracts nrType from var then reassigns the result to var (same as var = var - nrType)
? var *= nrType - multiplication assignement, multiplies var by nrType then reassigns the result to var (same as var = var * nrType)
? var /= nrType - division assignement, divides var by nrType then reassigns the result to var (same as var = var / nrType)
? var **= nrType - exponentiation assignement, takes var to the power of nrType then reassigns the result to var (same as var = var ** nrType)
? var %= nrType - modulo division assignement, modulo divides var by nrType then reassigns the result to var (same as var = var % nrType)
? var //= nrType - floor division assignement, floor divides var by nrType then reassigns the result to var (same as var = var // nrType)
? var &= int - bitwise and assignement, performs a bitwise and operation between var and int then reassigns the result to var (same as var = var & int)
? var |= int - bitwise or assignement, performs a bitwise or operation between var and int then reassigns the result to var (same as var = var | int)
? var ^= int - bitwise xor assignement, performs a bitwise xor operation between var and int then reassigns the result to var (same as var = var ^ int)
? var <<= int - bitwise left shift assignement, performs a bitwise left shift operation between var and int then reassigns the result to var (same as var = var << int)
? var >>= int - bitwise right shift assignement, performs a bitwise right shift operation between var and int then reassigns the result to var (same as var = var >> int)
Comparison Operators
? val == val - equality comparison, can compare any data types (primitive or complex)
- cannot compare objects or function definitions (always returns False)
- returns True for same data references (ex: for mutable data)
? val != val - unequality comparison, can compare any data types (primitive or complex)
- cannot compare objects or function definitions (always returns True)
- returns True for unequal data references (ex: for mutable data)
Bitwise Operators
? int & int - bitwise and, int are converted to binary (Python uses two's complement to convert to negative)
- in the result sets bits to 1 where both operands have 1 bits
? int | int - bitwise or, int are converted to binary (Python uses two's complement to convert to negative)
- in the result sets bits to 1 where any of the operands have 1 bits
? int ^ int - bitwise xor, int are converted to binary (Python uses two's complement to convert to negative)
- in the result sets bits to 1 where any of the operands have 1
- sets bits to 0 where both operands have 1 bits
? ~int - bitwise not, int is converted to binary (Python uses two's complement to convert to negative)
- flips the bits of the operand (1 to 0) (0 to 1)
? int << int - bitwise left shift, int left operand is converted to binary (Python uses two's complement to convert to negative)
- shifts bits to left by int (right operand) (basically adds 0 bits to the right)
? int >> int - bitwise right right, int left operand is converted to binary (Python uses two's complement to convert to negative)
- shifts bits to right by int (right operand) (basically removes bits from the right)

Operator Precedence

- precedence describes in which order operations are performed in a statement (ex: * operation has higher precedence than +) (equal operations are performed left-to-right) 

    100 + 50 * 3                                                                       // -> 250        // * operations performed before + 
    (100 + 50) * 3                                                                     // -> 450        // () operator has higher precedence than *
    100 + 50 - 3                                                                       // -> 147        // equal operations are performed from left-to-right

Bitwise Operators & / | / ^ / ~ / << / >>

- Python uses two's complement to represent negative integers in binary

- Python does not have >>> bitwise because integers are arbitrary large

DEMO 
// bitwise AND ------------------------------------------------------------------------
    21 & 68;                                                                           // -> 4 
        //               (Python binary representation)  
        // 21               ->   10101 
        // 68               -> 1000100 
        //  4               ->     100                                                 // sets the bits to 1 where both operands have 1 bits 
    
    -21 & -68;                                                                         // -> -88 
        //               (Python binary representation)  
        //  -21             -> 11101011                                                // -! Python uses two's complement to represent negative integers  
        //  -68             -> 10111100
        //  -88             -> 10101000
        
        
// bitwise OR -------------------------------------------------------------------------
    21 | 68;                                                                           // -> 85       
        //               (Python binary representation)  
        // 21               ->   10101 
        // 68               -> 1000100 
        // 85               -> 1010101                                                 // sets the bits to 1 where operands have 1 bits 
    
    
// bitwise XOR ------------------------------------------------------------------------           
    21 ^ 68;                                                                           // -> 81 
        //               (Python binary representation)  
        // 21               ->   10101 
        // 68               -> 1000100 
        // 81               -> 1010001                                                 // sets the bits to 1 where operands have 1 bits 
                                                                                       // sets the bits to 0 where both operands have 1 bits 
    
// bitwise NOT ------------------------------------------------------------------------  
    ~21                                                                                // -> -22 
        //               (Python binary representation)  
        //   21             ->    10101  
        //  -22             -> 11101010                                                // inverses the bits 
        
    ~68                                                                                // -> -69 
        //   68             ->  1000100                                 
        //  -69             -> 10111011                                                // inverses the bits 
    
    ~-341                                                                              // -> 340
        //  -341            -> 1010101011
        // 340              ->  101010100                                              // inverses the bits
    
    
// bitwise left shift -----------------------------------------------------------------
    21 << 1;                                                                           // -> 42
        //               (Python binary representation)  
        // 21               -> 10101 
        // 42               -> 101010                                                  // shift bits to left by 1, adds 0's on the right while shifting  
    
    68 << 3;                                                                           // -> 544
        // 68               -> 1000100 
        //                  -> 1000100000                                              // shift bits to left by 3, adds 0's on the right while shifting
        
    -152 << 7;                                                                         // -> -195456
        //  -152            -> 101000001001
        //  -195456         -> 1010000010010000000
        
        
// bitwise right shift ----------------------------------------------------------------
    21 >> 1;                                                                           // -> 10
        //               (Python binary representation)  
        // 21               -> 10101 
        // 10               -> 1010                                                    // shift bits to right by 1 (basically removes 1 bit from left)
    
    68 >> 3;                                                                           // -> 8
        // 68               -> 1000100 
        // 8                -> 1000                                                    // shift bits to rigght by 3 (basically removes 3 bits from right)
    
    -152 >> 7;                                                                         // -> -2
        //  -152            -> 101101000
        //  -2              -> 10
